# OpenAI Codex provider authentication

TradingAgents will add an experimental `openai-codex` provider for ChatGPT sign-in based Codex access, separate from the existing OpenAI API key provider. The immediate implementation will use Codex access tokens with the current OpenAI-compatible client path through a provider-specific bearer-auth path, because OpenAI documents Codex access tokens for Codex workflows rather than as Platform API keys. The provider will run a lightweight preflight model request before a full analysis so unsupported or expired credentials fail with a clear `openai-codex` message. The longer-term target is a Codex execution provider that delegates model work to official Codex execution surfaces. Token discovery will be best-effort: use `CODEX_ACCESS_TOKEN`, read file-backed Codex credentials when available, and guide users to file-backed Codex auth or explicit token export when credentials live only in an OS keyring.

OpenAI-family providers will support separate quick and deep reasoning effort defaults. `openai-codex` defaults to `gpt-5.5` with low effort for quick work and `gpt-5.5` with medium effort for deep work; the legacy `openai_reasoning_effort` setting remains as a backward-compatible fallback when the new quick/deep effort settings are unset.

Token setup will be available in both interactive and programmatic paths. The CLI will run a friendly setup helper during provider selection, while the lower-level client path will still resolve tokens and produce clear non-interactive errors for users who instantiate `TradingAgentsGraph` directly.
